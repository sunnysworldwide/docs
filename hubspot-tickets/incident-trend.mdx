# Get Incident Trend Statistics

Use this endpoint to retrieve **incident trend data** between two dates.  
You receive a list of date labels and a corresponding series of incident counts.

<Info>
This endpoint belongs to the **Stats** section.
Authentication is required.
</Info>

## Endpoint
```

GET /stats/incidents-trend/:startDate/:endDate

````

This endpoint returns the number of incidents created each day within the provided date range.

If no valid range is provided, the backend automatically normalizes the dates to a **one-week window starting today**.

## Authentication

```bash
Authorization: Bearer <your_access_token>
````

## Path Parameters

<ParamField type="string" required>
  Start date in `YYYY-MM-DD` format.\
  Used to define the beginning of the statistics window.
</ParamField>

<ParamField type="string" required>
  End date in `YYYY-MM-DD` format.\
  Used to define the end of the statistics window.
</ParamField>

<Note>
  Dates are inclusive. The backend adjusts invalid or empty date values into a correct range automatically.
</Note>

## Response Structure

<ResponseField name="labels" type="string[]">
  Array of date labels representing each interval day.
</ResponseField>

<ResponseField name="series" type="number[]">
  Array of incident counts aligned with the `labels` array.
</ResponseField>

---

## Example Request

<RequestExample>

```bash
curl -X GET "https://api.example.com/stats/incidents-trend/2025-01-01/2025-01-07" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

</RequestExample>
